Critical Findings (High Risk)

🔴 [HIGH] IDOR - Insecure Direct Object Reference on Expense Deletion

URL: DELETE /expenses/{id}

Description: The application does not verify that the authenticated user owns the expense being deleted. An attacker can modify the expense ID in the request to delete another user's expenses.

Solution: Add authorization check in the controller: $expense->user_id === auth()->id() or use Laravel Policy classes.

HIGH CWE-639 OWASP A01:2021

🔴 [HIGH] Missing Anti-CSRF Tokens on Budget AI Form

URL: POST /budget-ai/recommend

Description: The Budget AI recommendation form does not include a CSRF token, making it vulnerable to Cross-Site Request Forgery attacks. An attacker could force a logged-in user to make unintended API calls to the Gemini service.

Solution: Ensure @csrf directive is included in all forms and VerifyCsrfToken middleware is applied.

HIGH CWE-352 OWASP A01:2021

🔴 [HIGH] SQL Injection Risk in Category Filter

URL: GET /expenses?category=

Description: The category filter parameter appears to be concatenated into a query without proper parameterization in certain edge cases.

Solution: Use Eloquent query builder or parameterized queries exclusively. Never use DB::raw() with user input.
